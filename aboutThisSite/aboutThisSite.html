How I made this site:

This site is coded almost entirely with HTML(hyper text markup language) and CSS(cascading style sheets). HTML is used to control what content goes on a given site and CSS is used to control how that content looks. For example, if one wanted to add a title to a website, they would type it in between an opening and closing h1 tag in HTML. If one then wanted to change the color of that text, to blue for example, they would use CSS, probably saying something like “h1 {color: blue}”.

Because CSS is typically used for aesthetic purposes, one could code a website entirely out of HTML. But the resulting site would end up looking bare and may confuse the user. For example, the image below shows how the home page of this site may look without CSS.

By using CSS and HTML together, developers are able to create responsive and professional looking websites. I believe that my work on this site illustrates that and shows my proficiency in both HTML and CSS. But with that being said, there are still some things about this site that could use improvement. I discuss that below.

How this site could be improved:

1. Better coding practices:

The biggest problem with this site is the way that many elements are built on the measurement “em.” This measurement is the browser text size, typically 16 pixels, but can be changed by users depending on their preferences. On the website, I built lots of elements surrounding text in ems because I thought it would enable the website to be more responsive. I discovered, however, that in some cases, this can lead to problems. In the future, I will ensure to only use ems on text when it makes sense and use them sparingly as a measurement for non-text elements on the website.

Another coding problem this site has is that a lot of CSS is longer than it needs to be and is poorly organized. Also, the heads of the HTML files could be configured better. Both these problems shouldn’t be too difficult to solve and in the near future I will update the website with these changes.

2. Accessibility:

In the process of coding this website, I only very briefly consulted an online resource to help make the page accessible to all users. For this reason, I strongly suspect that there are major accessibility problems on the site. I am planning on fixing these issues in the near future.
